Adaptation d’un projet TIC-80 vers Solar2D

Introduction

Adapter un projet de TIC-80 (une fantasy console) vers Solar2D (anciennement Corona SDK) est tout à fait faisable, mais nécessite quelques ajustements, notamment au niveau des API graphiques et logiques.

Outils évalués

1. Windsurf

Windsurf est un outil de développement conçu spécifiquement pour Solar2D. Il permet une gestion de projet moderne, avec notamment :

  • L’autocomplétion pour les API Solar2D
  • L’organisation de projets multi-fichiers
  • La gestion simplifiée des assets et du code
Pertinent ? Oui, Windsurf est un bon choix pour développer en Solar2D, surtout si tu viens d’un environnement plus structuré comme TIC-80.

2. Cursor

Cursor est un éditeur de code basé sur AI (souvent intégré à GitHub Copilot ou similaire). Il est indépendant de Solar2D mais peut t’aider à :

  • Convertir du code TIC-80 (souvent en Lua, mais avec des particularités) vers du Lua standard
  • Faire des suggestions intelligentes pour le refactoring
Pertinent ? Oui, mais en complément. Cursor peut accélérer la traduction logique du code, mais ne remplace pas une compréhension des API spécifiques à Solar2D.

Adaptation TIC-80 → Solar2D : Points clés

  • Langage : TIC-80 utilise Lua aussi, mais avec des fonctions spécifiques (cls, spr, map, etc.). Il faut les mapper vers Solar2D (display.*, sprite, tilemap, etc.).
  • Graphismes : Pas de framebuffer direct dans Solar2D, il faut utiliser des display.newImageRect, SpriteSheet, etc.
  • Input : TIC-80 gère les touches avec btn(), Solar2D utilise Runtime:addEventListener("key", ...).
  • Boucle de jeu : TIC-80 a TIC(); Solar2D utilise enterFrame.

Recommandation finale

Utilise Windsurf comme base de développement Solar2D.
Utilise Cursor pour t’aider à convertir du code ou organiser tes scripts.
Prends le temps de mapper les fonctions de TIC-80 vers les équivalents Solar2D.
